## Recolección de Evidencias
Se realiza la recolección en un entorno controlado, asegurando que el proceso no altere los datos originales.
Se documenta cada acción y se siguen protocolos establecidos para la obtención y registro de las evidencias digitales.
Todas las operaciones se ejecutan conforme a procedimientos verificados, manteniendo la trazabilidad y la integridad de la información.

## Descripción de la Evidencia
Se recopilan elementos digitales relevantes para el proceso de análisis forense.
Cada evidencia es identificada, clasificada y documentada según su tipo y propósito.
Se asegura la correcta gestión de la información, preservando su autenticidad y valor probatorio.

## Cadena de Custodia
Se establece una cadena de custodia que detalla cada manipulación, traslado o almacenamiento de las evidencias.
Se registran fechas, responsables y condiciones de manejo para mantener la trazabilidad de todo el proceso.
La documentación asociada garantiza la transparencia y validez legal de las actuaciones efectuadas.

##  Almacenamiento de la Evidencia
Las evidencias se almacenan en medios seguros y bajo control de acceso restringido.
Se crean copias de respaldo y se aplican medidas de protección que aseguran integridad, confidencialidad y disponibilidad.
Se realizan verificaciones periódicas para confirmar la conservación y fidelidad del material almacenado.

##  Metodología Aplicada
La metodología empleada se basa en prácticas internacionales para la gestión de evidencia digital.
Se contemplan fases de identificación, recolección, preservación y documentación sistemática.
El objetivo principal es garantizar la validez técnica y legal del proceso, así como la posibilidad de reproducirlo de manera verificable.
Marco normativo de referencia: ISO/IEC 27037 para el manejo de evidencia digital y NIST SP 800-86 para la integración del análisis forense en el ciclo de respuesta a incidentes.

# Proceso de captura de evidencias

- EV01 - Máquina virtual OVA:  Se aisló el activo en entorno controlado, se exportó la máquina virtual a formato OVA de forma reproducible, se calcularon y registraron MD5, SHA-1 y SHA-256, se verificó la integridad tras la transferencia y se preservó en almacenamiento cifrado con trazabilidad en la cadena de custodia.

- EV02 - Memoria RAM (.elf):  Se pausó la máquina virtual para congelar el estado, se obtuvo el volcado de memoria desde el hipervisor, se calcularon y registraron MD5, SHA-1 y SHA-256, se comprobó la integridad tras la copia y se guardó en almacenamiento cifrado manteniendo registro completo en la cadena de custodia.

- EV03 - Disco virtual (imagen): Se generó una imagen bit a bit del disco virtual utilizando FTK Imager, documentando metadatos de la adquisición, calculando y registrando MD5, SHA-1 y SHA-256, verificando la integridad tras la transferencia y preservando la imagen en almacenamiento cifrado con trazabilidad completa en la cadena de custodia.

- EV04 - Logs y triage_01: Se ejecutó el comando Windows “systeminfo” para volcar configuración y estado del sistema, incluyendo nombre de host, versión de OS y Service Pack, fabricante/modelo, arquitectura, fecha de instalación, último arranque, BIOS, rutas de sistema, zona horaria, memoria física y virtual, dominio/inicio de sesión, hotfixes instalados y configuración de red con IP/DHCP, quedando todo registrado en el archivo adjunto.

- EV05 - Logs y triage_02: Se ejecutó el comando “ipconfig /all” para obtener la configuración completa de red del equipo, incluyendo nombre de host, tipo de nodo, estado de enrutamiento/WINS, adaptadores y descripciones, MAC, estado DHCP, direcciones IPv4/IPv6, máscara, puerta de enlace, servidor DHCP, tiempos de concesión, servidores DNS y parámetros de túneles ISATAP/6to4/Teredo, quedando todo registrado en el archivo correspondiente.

- EV06 - Logs y triage_03: Se ejecutó el comando “route print” para listar la tabla de rutas IPv4 e IPv6 del equipo, incluyendo interfaces disponibles, rutas activas con destino, máscara, puerta de enlace, interfaz y métrica, así como rutas persistentes, permitiendo identificar gateways, redes locales, rutas por defecto y túneles como ISATAP/6to4/Teredo

- EV07 - Logs y triage_04: Se ejecutó el comando “arp -a” para listar la caché ARP de la interfaz activa, mostrando correspondencias IP–MAC y su tipo (dinámica/estática), lo que permite identificar puertas de enlace, equipos vecinos y direcciones de difusión o multicast resolvidas recientemente.

- EV08 - Logs y triage_05: Se ejecutó el comando “netstat -ano” para enumerar conexiones y puertos abiertos con protocolo, direcciones local/remota, estado (LISTENING/ESTABLISHED/SYN_SENT), y PID asociado, abarcando TCP/UDP en IPv4 e IPv6; esta salida permite correlacionar puertos con procesos y detectar actividad de red relevante en el momento de la captura

- EV09 - Logs y triage_06: Se ejecutó el comando “tasklist /v” para enumerar procesos en ejecución con detalle extendido, incluyendo nombre de imagen, PID, sesión y número de sesión, uso de memoria, estado, usuario propietario, tiempo de CPU y título de ventana, lo que facilita correlacionar procesos con actividad de red, servicios y sesiones interactivas.

- EV10 - Logs y triage_07: Se ejecutó el comando “wmic process list full” para obtener propiedades detalladas de cada proceso, incluyendo nombre, PID, ruta del ejecutable, línea de comandos, usuario/sesión, proceso padre, prioridad, consumo de memoria, contadores de E/S y tiempos de CPU, permitiendo un inventario completo y atributos útiles para correlación con eventos y red.

- EV11 - Logs y triage_08: Se ejecutó el comando “schtasks /query /v /fo list” para enumerar tareas programadas con detalle, incluyendo nombre, estado, última y próxima ejecución, autor, acción a ejecutar (Task To Run), usuario de ejecución, carpeta, tipo de programación y parámetros de repetición, permitiendo identificar tareas legítimas y potenciales persistencias en el sistema.

- EV12 - Logs y triage_09: Se ejecutó el comando “query user” para listar las sesiones activas del sistema, mostrando usuario, nombre de sesión, ID de sesión, estado, tiempo de inactividad e instante de inicio de sesión, útil para determinar quién está conectado y desde cuándo.

- EV13 - Logs y triage_10: Se ejecutó el comando “set” para volcar las variables de entorno del sistema y del usuario actual, incluyendo rutas críticas como Path, SystemRoot y windir, perfiles y rutas de datos (USERPROFILE, APPDATA, TEMP/TMP), arquitectura y CPU, y valores de sesión, útiles para contextualizar ejecuciones, localizar binarios y comprender el entorno de ejecución.
